      • post /oauth2/token/{grant_type}

        • Get access token

        Implementation Notes

        Creating client id/secret
        Before you can authorize with the API you’ll need to set the client id and the secret on REST API admin page.
        With the client id and client secret you can request an access token using the POST api/rest/oauth2/token api call.

        Authenticate API requests with access token
        The received token needs to be added to every API call.
        Include an Authorization header with the following value: Bearer .
        For example: Authorization: Bearer 7487c50afc37dbf6e5a484dc49c363128036c356

        Service urls:
        http://yourdomain.com/api/rest/oauth2/token/client_credentials
        OR
        http://yourdomain.com/index.php?route=feed/rest_api/gettoken&grant_type=client_credentials

        Response Class

        string


        Parameters

        Parameter Value Description Parameter Type Data Type
        Authorization Authorization header value.
        How to create Authorization header value:
        Concatenate the client id, a colon character ":",and the client secret into a single string. So you’ll get client_id:client_secret.
        Base64 encode the concatenated string
        Add "Basic " (including the space) before the encoded string.
        If your client id is demo_oauth_client and your client secret is demo_oauth_secret, your concatenated string will be demo_oauth_client:demo_oauth_secret.
        After Base64 encode you will get ZGVtb19vYXV0aF9jbGllbnQ6ZGVtb19vYXV0aF9zZWNyZXQ= and the header parameter will be
        Basic ZGVtb19vYXV0aF9jbGllbnQ6ZGVtb19vYXV0aF9zZWNyZXQ=                 		header string
        grant_type Grant type path string
        Old token
        		If given, the users data (ie shopping cart products) is transfered from this access token to the newly generated access token. This is usefull when the old access token expired and you have to request a new access token but want to retain the shopping cart products and other user related data.
        Sample old token: 
        {
        	"old_token":"old_access_token"
        }
